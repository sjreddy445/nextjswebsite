    Part 1 | A Glimpse of JavaScript in the backend – nodeJS

    In our first tech-blog series, our developer, Akhilesh Kumar shares details on nodeJS – what its all about and what it’s not. Several platforms like WhatsApp, Facebook, Google and many more are currently drawing benefits from nodeJS. So, tech-geeks take a note and read-on to know more!

    What is nodeJS 
    In layman’s word, nodeJS is JavaScript run-time environment for executing JavaScript code on server-side similar as JRE and it is mostly used to make web-servers/application servers like Apache Tomcat, IIS, Jetty, Glassfish etc.

    It executes JavaScript in the server side. Yes, JavaScript without a browser. nodeJS itself is not made in JavaScript completely, rather it’s wrappers are made in C. It just executes JavaScript.

    What nodeJS is NOT
    ■    nodeJS is not a framework, it’s a server
    ■    nodeJS wrappers are written in C and not in JavaScript which works on top of V8 engine
    ■    It’s not multi-threaded; It runs in a single thread with callback concept

    What nodeJS is
    ■    It is a server which can interpret JavaScript.
    ■    It is a open source and cross platform to make real time network applications.
    ■    It provides you asynchronous, event driven I/O APIs.
    ■    It runs single threaded event based loops, so all executions become non-blocking.

    Architecture

    nodeJS has several layers in its architecture written in JavaScript and C/C++.

    First layer has all standard libraries written in JavaScript which generally interacts to our application. Then we have several node bindings(libUv) then after inner layers.

    LibUv is an abstraction layer on the top of libeio , libev, c-ares ( for DNS ) and iocp (for windows asyncronous-io). LibUv performs, maintains and manages all the io and events in the event pool. ( in case of libeio thread-pool ).

    Crypto helps to implement cryptography to perform encryption and decryption on root level.

    DNS helps us on resolving domains on root level.

    Libev is the event loop which actually runs internally in node.js to perform simple event loop operations. It’s written originally for *nix systems. Libev provides a simple yet optimized event loop for the process to run on.

    LibEio is a library to perform input output asynchronously. It handles file descriptors, data handlers, sockets etc.

    V8 Engine help us to execute JavaScript logics and converting them to machine code.
